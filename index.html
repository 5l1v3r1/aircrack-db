Notes:

1: May be known as TET-N150 as I boght it from Tehnoetic.
2: The 5GHz frequency band is for IEEE 802.11a mode only. It can read beacons sent by IEEE 802.11n/ac networks, but the packet injection won't work for these modes. Packet injection for IEEE 802.11n works only on 2.4GHz. With the 3.17-rc3-1 driver it can capture WPA handshakes on 5GHz, but it can run only as a completely passive attack.
3: The adapter was sold without casing. It is only the board with a soldered USB port. While the OUI indicates B-Link as manufacturer, it is unrelated to 8level WUSB-150A.
4: The monitoring mode on Broadcom STA / wl may be enabled via arcane methods described in this document. The card is installed into a MacBook Air 6,2. I'll test it as soon as the UEFI boot is fixed with Kali as it is broken since the OS X 10.9.5 update.
5: The GPL driver supplied by Realtek doesn't seem to implement the monitor mode.
6: Tested under Debian 6 with compat-wireless 2.6.39-1 and the ar9170usb driver. The 5GHz support is the same as AWUS051NH. Some patches needed to build this driver. ar9170usb was dropped from the mainline kernel since 3.0 in favor of carl9170 which badly suports monitor mode and only monitor mode. You don't get packet injection with carl9170.
7: Both cards are equally capable for injecting packets. While AR9331 supports 802.11n, the RTL8187L interface has better TX power (27 dBm vs. 18 dBm).
8: Tested against an ALFA AWUS036H connected via USB. Running OpenWrt 14.07 (Brarrier Breaker).
9: Tested under Trisquel 6 with a trunk build of aircrack-ng.
